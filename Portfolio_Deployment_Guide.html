PHASE 1 — Prepare Your Code Locally   (~10 minutes)

1Open PowerShell in the project folder

Press Win + X then click Terminal or Windows PowerShell. Then type:

cd "c:\Users\desai\OneDrive\Desktop\1_Resume_InterviewPrep\AbhishekPortfolio"

Keep this PowerShell window open — you will use it for all commands below.

2Confirm these files already exist in the project

The following files were already created and are ready:

  • .gitignore — tells Git to ignore build folders and secrets
  • README.md — description page shown on GitHub
  • api\appsettings.json — connection string and admin password are already set to placeholders (real values go into Azure later)
Tip: Open api\appsettings.json and confirm the connection string says CHANGE_IN_AZURE_APP_SETTINGS — not a real password. This is intentional.

3Do a final check — make sure no sensitive data is going to GitHub

In your PowerShell window run:

Get-Content "api\appsettings.json"

You should see CHANGE_IN_AZURE_APP_SETTINGS for both the connection string and AdminPassword. If you see a real password, change it to that placeholder before continuing.

4Build and confirm the API compiles cleanly

cd api
dotnet build
cd ..

You should see Build succeeded with 0 errors. If there are errors, fix them before continuing.

PHASE 2 — Push to GitHub   (~10 minutes)

5Create a free GitHub account (skip if you have one)

  1. Go to github.com
  2. Click Sign up
  3. Use any email address, choose a username (e.g. abhishekdesai-dev)
  4. Verify your email

6Initialize Git and push your code

Part A — Initialize Git locally (run each line separately in PowerShell):

cd "c:\Users\desai\OneDrive\Desktop\1_Resume_InterviewPrep\AbhishekPortfolio"
git init
git add .
git status

After git status, review the list of files. You should NOT see any bin\, obj\, or password files. If anything looks wrong, stop and ask before continuing.

git commit -m "Initial commit: ASP.NET Core 10 portfolio API + AngularJS frontend + SQL schema"

Part B — Create the GitHub repository online:

  1. Go to github.com → click the + icon top-right → New repository
  2. Repository name: abhishek-portfolio
  3. Visibility: Public (recruiters must be able to see it — private repos are invisible)
  4. Do NOT check "Add a README file" — you already have one
  5. Click Create repository

Part C — Connect and push (copy your GitHub username into the command below):

git remote add origin https://github.com/YOUR_GITHUB_USERNAME/abhishek-portfolio.git
git branch -M main
git push -u origin main

GitHub will ask for your username and password. For password, use a Personal Access Token (not your GitHub login password). To get one: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → check repo scope → copy the token → paste it as the password.

Done! Open https://github.com/YOUR_USERNAME/abhishek-portfolio — you should see all your files and the README.

PHASE 3 — Create Azure SQL Database (Free)   (~20 minutes)

7Create a free Azure account

  1. Go to portal.azure.com
  2. Click Start free or sign in if you already have an account
  3. Free account gives you $200 credit for 30 days + certain services free forever
  4. Azure SQL Database has a free offer — 100,000 vCore seconds/month, no charge
No credit card charges: The free tier App Service (F1) and free-offer Azure SQL Database are $0/month. You will be asked to enter a credit card when creating the account but you will NOT be charged as long as you stay within free tier limits.

8Create a SQL Server (the container for your database)

  1. In the Azure Portal search bar at the top, type: SQL servers → click it
  2. Click + Create
  3. Fill in these fields:
    FieldValue
    SubscriptionYour subscription (e.g. "Free Trial" or "Pay-As-You-Go")
    Resource groupClick Create new → type portfolio-rg → OK
    Server nameabhishek-portfolio-sql (must be globally unique — add numbers if taken, e.g. abhishek-portfolio-sql-2025)
    RegionEast US (or nearest to you)
    Authentication methodUse SQL authentication
    Server admin loginportfolioadmin
    PasswordPick a strong password — write it down! You will need it in Step 13.
  4. Click Review + create → Create
  5. Wait ~1-2 minutes

9Create the Database with the FREE offer

  1. In the Azure Portal search bar, type: SQL databases → click it
  2. Click + Create
  3. Fill in these fields:
    FieldValue
    SubscriptionYour subscription
    Resource groupportfolio-rg
    Database nameAbhishekPortfolioDB
    ServerSelect abhishek-portfolio-sql (the one you just created)
    Want to use SQL elastic pool?No
    Workload environmentDevelopment
  4. Under Compute + storage, click Configure database
    • Look for the banner: "Apply the Azure SQL Database free offer" — click it
    • If you see it, select it. This gives you free database forever (100K vCore-seconds/month)
    • If you do NOT see the free offer: choose Serverless tier, set max vCores to 1 — this costs nearly $0 for low traffic
  5. Click Review + create → Create
  6. Wait ~3-5 minutes for deployment

10Allow your PC and Azure services to connect to the database

  1. Go to your SQL Server resource: search SQL servers → click abhishek-portfolio-sql
  2. In the left menu, click Networking (under Security)
  3. Under Firewall rules:
    • Click "Add your client IPv4 address" — this lets your PC connect
    • Toggle "Allow Azure services and resources to access this server" to Yes — this lets your App Service connect
  4. Click Save

Now run the SQL script to create all tables and seed data:

  1. In Azure Portal → go to your SQL Database (SQL databases → AbhishekPortfolioDB)
  2. In the left menu, click Query editor (preview)
  3. Login with: portfolioadmin and the password you wrote down in Step 8
  4. Open the file sql\AbhishekPortfolio_DB_Setup.sql on your computer in Notepad
  5. Select all text (Ctrl+A), copy (Ctrl+C)
  6. Paste it into the Query editor (Ctrl+V)
  7. Click Run
  8. You should see: "Query succeeded" or a list of table creation messages
Done! Your database is live in Azure with all tables and data.

PHASE 4 — Deploy API to Azure App Service (Free F1 Tier)   (~20 minutes)

11Create the App Service (web hosting for your API)

  1. In Azure Portal search bar, type: App Services → click it
  2. Click + Create → Web App
  3. Fill in these fields:
    FieldValue
    SubscriptionYour subscription
    Resource groupportfolio-rg
    Nameabhishek-portfolio (this becomes your URL — must be globally unique)
    PublishCode
    Runtime stack.NET 10 (LTS)
    Operating SystemWindows
    RegionEast US
  4. Under Pricing plans → click Create new
    • Give it any name
    • Click Change size → click Dev / Test tab → select F1 (Free) → click Apply
  5. Click Review + create → Create
  6. Wait ~2-3 minutes

12Get your Azure SQL connection string

  1. Go to Azure Portal → SQL databases → AbhishekPortfolioDB
  2. In left menu, click Connection strings
  3. Click the ADO.NET tab
  4. Copy the connection string — it looks like:
    Server=tcp:abhishek-portfolio-sql.database.windows.net,1433;Initial Catalog=AbhishekPortfolioDB;Persist Security Info=False;User ID=portfolioadmin;Password={your_password};MultipleActiveResultSets=False;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;
  5. Replace {your_password} with the actual password you set in Step 8
  6. Keep this copied — you will use it in the next step

13Set environment variables in Azure (your real secrets go here — NOT in code)

  1. Go to Azure Portal → App Services → abhishek-portfolio
  2. In left menu, click Environment variables (under Settings)
  3. Click + Add for each row below:
NameValue
ConnectionStrings__DefaultConnectionThe full connection string from Step 12 (with real password)
AdminPasswordA strong password you will use to log into /admin.html (example: MyPortfolio2025!)
ASPNETCORE_ENVIRONMENTProduction
Note the double underscore in ConnectionStrings__DefaultConnection — that is how .NET reads nested config from environment variables. One underscore will not work.
  1. Click Apply at the bottom
  2. Click Confirm in the popup

14Connect GitHub to Azure for automatic deployment

  1. In your App Service, left menu → click Deployment Center (under Deployment)
  2. Under Source, select GitHub
  3. Click Authorize → a GitHub login popup appears → sign in → click Authorize AzureAppService
  4. Back in Azure:
    • Organization: your GitHub username
    • Repository: abhishek-portfolio
    • Branch: main
  5. Click Save

Azure will now:

  • Add a GitHub Actions workflow file to your repo automatically
  • Trigger the first deployment (takes 3–5 minutes)
  • Every time you do git push in the future, it auto-deploys
Important — Tell Azure where your project file is: Your .csproj is inside the api/ subfolder, not the root. After Azure creates the workflow file, you may need to edit it. Go to your GitHub repo → .github/workflows/ → open the .yml file → find the line with dotnet publish and make sure it points to api/AbhishekPortfolio.Api.csproj. If deployments fail, this is the first thing to check.

PHASE 5 — Verify Everything is Live   (~5 minutes)

15Watch the deployment

  1. Go to your GitHub repo → click the Actions tab at the top
  2. You will see a workflow running (orange spinning circle)
  3. Click on it to see live build logs
  4. When it turns green (✓), the deployment is done
  5. If it turns red (✗), click on it and read the error — most common issue is the project path (see warning in Step 14)

16Open your live URLs

WhatURLExpected result
Portfolio Homepagehttps://abhishek-portfolio.azurewebsites.netYour portfolio site loads with all data
Swagger API Docshttps://abhishek-portfolio.azurewebsites.net/swaggerSwagger UI shows all 15 endpoints
Admin Dashboardhttps://abhishek-portfolio.azurewebsites.net/admin.htmlLogin screen appears — use the AdminPassword you set in Step 13
If the site loads but data is missing: The API connected but the SQL tables may be empty. Make sure the SQL script ran successfully (Step 10). Also check App Service → Log stream (under Monitoring) for any database connection errors.
